# ROAM Pages and UI Requirements

## Public Pages
- Landing page.
- Login page.
- Sign up page.

## Authenticated Shipper Pages
- Shipper dashboard.
- Post new load page.
- My loads page.
- Load detail page with bid list.
- Notifications page.
- Account settings page.

## Authenticated Driver Pages
- Driver dashboard.
- Open loads page.
- Load detail page.
- My bids page.
- Assigned loads page.
- Notifications page.
- Account settings page.

## Admin Pages
- Admin dashboard.
- Users table.
- Loads table.
- Bids table.

## UI Principles
- Mobile responsive first.
- Clean marketplace layout, not a corporate brochure.
- Dashboard should be simple and easy to scan.
- Status badges should be highly visible.
- Primary actions should be obvious: Post Load, Place Bid, Accept Bid.

## Key Components
- Header / top nav.
- Sidebar or mobile nav.
- Status badges.
- Load cards.
- Bid cards.
- Empty states.
- Notification list.
- Modal or confirmation dialog for accepting a bid.

## Important Empty States
- No open loads yet.
- No bids yet on this load.
- No assigned loads yet.
- No notifications yet.

## Suggested Dashboard Sections

### Shipper Dashboard
- Open loads.
- Loads awaiting bids.
- Recent bids.
- Assigned loads.

### Driver Dashboard
- Nearby open loads.
- My pending bids.
- Accepted assignments.
- Recent notification activity.
